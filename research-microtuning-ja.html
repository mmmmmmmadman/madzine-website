世界の微分音システム研究

Microtuning Systems of the World: A Technical Analysis for Digital Implementation

MADZINE 2024-12 Version 1.0

Abstract

本論文は、世界各地の伝統音楽における微分音（microtuning）システムを研究し、その音程構造、文化的背景、およびデジタル音楽シンセサイザーでの実装に関する技術的考察を行う。特に、2オクターブの独立した微分音設定が必要なシステム、および上行・下行音階の差異に焦点を当てる。

1. 序論

1.1 微分音とは

微分音（Microtone）とは、西洋の十二平均律（12-TET）の半音（100セント/cents）よりも小さい音程を指す。世界各地の伝統音楽において、多くの文化が独自の音階システムを発展させており、その音程は西洋の標準に従わない。

1.2 セント（Cents）システム

セントは音程を測定するための対数単位である：

  • 1オクターブ = 1200 cents
  • 1半音（12-TET）= 100 cents
  • 1セント = 1/100 半音

計算式：

cents = 1200 × log₂(f₂/f₁)

1.3 研究目的

本研究の目的：

  1. 世界の主要な微分音システムを体系的に整理する
  2. 各システムの音程構造を分析する
  3. 2オクターブの独立設定が必要なシステムを特定する
  4. VCV Rack Quantizer モジュールの実装根拠を提供する

3. 中東の音階システム

3.1 アラビアの Maqam システム

3.1.1 概要

Maqam（複数形 Maqamat）はアラビア音楽の旋法システムであり、24-TET を近似として使用する。伝統的に24の音は2オクターブにまたがり、Yakah（低いG）から Ramal Tuti（高いG）までの範囲をカバーする。

3.1.2 主要な Maqam とその音程

Maqam Rast（ラスト）

音名：C  D  E½♭  F  G  A  B½♭  C
音程：0  200  350  500  700  900  1050  1200

E½♭ と B½♭ は約50セント低い音を表す。

Maqam Bayati（バヤーティー）

音名：D  E½♭  F  G  A  B♭  C  D
音程：0  150  300  500  700  850  1000  1200

Maqam Hijaz（ヒジャーズ）

音名：D  E♭  F#  G  A  B♭  C  D
音程：0  100  400  500  700  800  1000  1200

3.1.3 2オクターブ間の差異

一部の Maqam では、上のオクターブの音程が下のオクターブと異なる場合がある：

Maqam Sikah の例：

  • 下のオクターブ：E½♭ - F - G - A - B½♭ - C - D
  • 上のオクターブ：異なる四分音の位置を使用する可能性がある

2オクターブ必要：はい（一部の Maqam）

3.2 トルコの Makam システム

3.2.1 概要

トルコの Makam システムは 53-TET（53等分律）を使用し、オクターブを53のコンマに分割する。

  • 長二度 = 9コンマ（約204 cents）
  • 短二度 = 4コンマ（約90 cents）
  • 1コンマ ≈ 22.6 cents

3.2.2 浮動音の概念

トルコ音楽の重要な特徴は「浮動音」（floating notes）である：

  • 上行時：特定の音がわずかに上がる（約1コンマ）
  • 下行時：同じ音がわずかに下がる

Makam Rast の例：

上行：G - A - B(+1) - C - D - E - F#(Irak) - G
下行：G - F(Acem) - E - D - C - B(-1) - A - G

注：上行では F#（Irak）を使用し、下行では F（Acem）を使用する。

3.2.3 Seyir（旋律の方向性）

Makam には3種類の旋律方向がある：

  1. Cinici（上行型）：主音から上方へ
  2. Inici（下行型）：高音から下方へ
  3. Inici-Cinici（混合型）：両者の組み合わせ

2オクターブ必要：はい（上行・下行の差異）

3.3 ペルシアの Dastgah システム

3.3.1 概要

ペルシア音楽は Dastgah（旋法システム）を使用し、7つの主要 Dastgah と5つの補助 Avaz を含む。

3.3.2 音程の特徴

ペルシア音楽の特徴的な音程：

  • Koron（↓）：約50-60セント低下
  • Sori（↑）：約50-60セント上昇

Dastgah Shur の例：

G - A(koron) - B♭ - C - D - E♭ - F - G
音程：0 - ~150 - 300 - 500 - 700 - 800 - 1000 - 1200

3.3.3 上行・下行の差異

トルコのシステムと同様に、ペルシア音楽にも上行・下行の微妙な差異がある。

2オクターブ必要：はい

4.1.4 Aroha と Avaroha

インドの Raga の最も重要な特徴は、Aroha（上行）と Avaroha（下行）でまったく異なる音を使用できることである。

Raga Yaman の例：

Aroha（上行）：N S G M# D N S'
Avaroha（下行）：S' N D P M# G R S

注：上行では Re(R) と Pa(P) が省略され、下行ではすべての音が使用される。

Raga Bhairav の例：

Aroha：S r G M P d N S'
Avaroha：S' N d P M G r S

Komal Re(r) と Komal Dha(d) を使用する。

4.1.5 3つのオクターブ

インド音楽は3つのオクターブにまたがる：

  • Mandra Saptak（低オクターブ）
  • Madhya Saptak（中オクターブ）
  • Tar Saptak（高オクターブ）

2オクターブ必要：はい（Aroha/Avaroha の差異 + 異なるオクターブで微妙な差異がある可能性）

5. 東アジアの音階システム

5.1 日本の音階システム

5.1.1 上原六四郎の理論（1895）

明治時代、上原六四郎は『俗楽旋律考』において以下を提唱した：

  1. 俗楽の音階は五音音階である
  2. 半音を含む陰旋と半音を含まない陽旋に分かれる
  3. いずれも上行形と下行形を持つ

5.1.2 主要な音階の種類と上行・下行の差異

都節音階（Miyako-bushi）- 陰旋法

下行的性格を持ち、上行時に音が変化する：

下行形（基本）：E - F - A - B - C - E
上行形（異向形）：E - F - A - B - D - E
                              ↑
                     短6度(C)→短7度(D)

律音階（Ritsu）

同様に下行的性格を持つ：

下行形：6度音は A♭ を使用
上行形：6度音が B♭ に上昇

陽音階（Yo Scale）

音名：D - E - G - A - B
音程：0 - 200 - 500 - 700 - 900

雅楽（Gagaku）に使用される。上行的性格を持ち、比較的安定している。

民謡音階（Min'yo）

上行的性格を持ち、中間音の浮動が少なく、上行・下行の差異は小さい。

5.1.3 小泉文夫の理論

小泉文夫は日本の音階を以下のように分類した：

  • 下行性音階：都節、律（上行時に音が変化する）
  • 上行性音階：民謡、琉球（比較的安定）

核心的な概念はテトラコルド（四度音列）であり、完全四度内の中間音が旋律の方向によって浮動する。

5.1.4 微分音の特徴

箏（Koto）の調律は純正律を用いる傾向がある：

  • 長三度：386 cents（純正律） vs 400 cents（12-TET）
  • 差異は約14セント

2オクターブ必要：はい（都節・律音階に上行・下行の差異がある）

5.2 琉球音階（Ryukyu Scale）

5.2.1 構造

五声版：C - E - F - G - B
音程：0 - 400 - 500 - 700 - 1100
構造：長三度 - 短二度 - 長二度 - 長三度 - 短二度

六声版：C - D - E - F - G - B
音程：0 - 200 - 400 - 500 - 700 - 1100

5.2.2 上行・下行の差異

研究によると、琉球音階は下行時に経過音「レ」（D）が加わる：

上行：C - E - F - G - B - C（五声）
下行：C - B - G - F - E - D - C（D を経過音として追加）

「てぃんさぐぬ花」などの楽曲において、「レ」は主に下行時に旋律をより滑らかにするために用いられる。

5.2.3 特徴

  • 特徴的な音程：B→C（短二度）は沖縄音楽を識別する鍵となる
  • 陽音階と関連するが、音高が上昇している
  • 八重山民謡には微妙な音高の変化がある
  • 小泉理論における「上行性音階」に属する

2オクターブ必要：はい（下行時に経過音が加わる）

5.3 中国の音階システム

5.3.1 五声音階（宮商角徴羽）

宮：C - D - E - G - A
音程：0 - 200 - 400 - 700 - 900

5.3.2 微分音

中国の伝統音楽は主に純正律を使用しており、特に古琴音楽には複雑な倍音体系がある。

2オクターブ必要：いいえ

6. 東南アジアの音階システム

6.1 インドネシアのガムラン（Gamelan）

6.1.1 Slendro 音階

五声音階で、音程はほぼ均等（各段約240セント）：

近似音程：0 - 240 - 480 - 720 - 960

ただし実際には、各ガムラン楽器セットが独自の調律を持ち、標準化されていない。

6.1.2 Pelog 音階

七声音階で、音程は不均等：

一般的な形式：0 - 120 - 270 - 540 - 670 - 800 - 1050

非常に狭い音程と非常に広い音程が混在するのが特徴である。

6.1.3 微分音の特徴

  • 各ガムランセットが独自の調律を持つ
  • Ombak（うなり）：対になった楽器を意図的にわずかにずらして調律し、うなり効果を生み出す
  • 西洋の 12-TET との差異は最大 ±50 cents に達する

2オクターブ必要：いいえ（ただし高度に個別化されている）

6.2 タイの音階

6.2.1 七平均律

タイの伝統音楽は 7-TET（七等分律）を使用する：

音程：0 - 171 - 343 - 514 - 686 - 857 - 1029
各段約 171 cents

2オクターブ必要：いいえ

7.2 実装方式の比較

方式A：24の微分音パラメータ（2オクターブ）

enum MicrotuneParam {
    // 第1オクターブ
    C1_MICROTUNE, CS1_MICROTUNE, D1_MICROTUNE, ..., B1_MICROTUNE,
    // 第2オクターブ
    C2_MICROTUNE, CS2_MICROTUNE, D2_MICROTUNE, ..., B2_MICROTUNE,
};

利点：

  • すべてのシステムを完全にサポート
  • 上下のオクターブを独立して制御

欠点：

  • 24個のノブが必要（UIスペースの問題）
  • 2オクターブが不要なシステムでは冗長

方式B：12 + 12（上行・下行を分離）

struct MicrotunePreset {
    float ascending[12];   // 上行の微分音
    float descending[12];  // 下行の微分音
};

利点：

  • 中東・インドシステムの要件に正確に対応
  • 音高の移動方向に応じて自動切り替え可能

欠点：

  • 音高の移動方向を追跡する必要がある
  • 実装がより複雑

方式C：ハイブリッド方式（推奨）

struct MicrotuneSystem {
    float base[12];        // 基本微分音（1オクターブ）
    float octave2[12];     // 第2オクターブ（オプション）
    bool useTwoOctaves;    // 2オクターブを有効にするか
    bool useAscDesc;       // 上行・下行を区別するか
    float ascending[12];   // 上行補正（オプション）
    float descending[12];  // 下行補正（オプション）
};

利点：

  • すべてのシステムを柔軟にサポート
  • 既存の12音システムとの後方互換性
  • プリセットに応じて自動設定可能

7.4 音高方向追跡アルゴリズム

class PitchDirectionTracker {
    float lastPitch = 0.f;
    bool ascending = true;

    void update(float currentPitch) {
        if (currentPitch > lastPitch + threshold) {
            ascending = true;
        } else if (currentPitch < lastPitch - threshold) {
            ascending = false;
        }
        lastPitch = currentPitch;
    }

    float getMicrotune(int note, const MicrotuneSystem& sys) {
        if (sys.useAscDesc) {
            return ascending ? sys.ascending[note] : sys.descending[note];
        }
        return sys.base[note];
    }
};

8. 結論と提言

8.1 主要な発見

  1. 中東および南アジアのシステムは2オクターブまたは上行・下行の区別が必要
  2. 東アジアのシステム（日本、中国、琉球）は2オクターブを必要としない
  3. 実装の複雑さ：ハイブリッド方式により、単純な要件と複雑な要件の両方を同時に満たすことができる

8.2 推奨する実装ステップ

  1. Phase 1：既存の12の微分音パラメータを維持
  2. Phase 2：「Two-Octave Mode」スイッチと第2組の12パラメータを追加
  3. Phase 3：「Ascending/Descending」モードと音高方向追跡を追加
  4. Phase 4：完全なプリセットライブラリを実装

8.3 プリセットライブラリの提案

優先的に実装すべきもの：

  1. Equal Temperament（12-TET）
  2. Just Intonation
  3. Pythagorean
  4. Arabic Maqam Rast
  5. Arabic Maqam Bayati
  6. Turkish Makam Rast（上下行を含む）
  7. Indian Shruti（22音の近似）
  8. Gamelan Slendro
  9. Gamelan Pelog
  10. Japanese In/Yo
  11. Ryukyu
  12. Quarter Tone
