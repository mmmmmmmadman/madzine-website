WorldRhythm：異文化リズム生成の統一フレームワーク

キーワード：リズム生成、計算民族音楽学、異文化音楽、インターロックパターン、ポリリズム、アルゴリズム作曲、装飾技法

本論文では、統一アーキテクチャ内で10の異なる音楽文化にわたるリズムパターンを生成するルールベースのアルゴリズムフレームワーク、WorldRhythmを紹介する。単一の文化的伝統に焦点を当てるか、西洋偏重のデータセットに依存する機械学習に頼る既存のアプローチとは異なり、WorldRhythmは4層の役割階層を採用し、文化固有の優先度マトリクス、独自のリズム概念のための専門エンジン、およびスタイル固有の装飾のためのArticulation Profileシステムを組み合わせている。このシステムは、西アフリカのポリリズム、アフロキューバンのクラーベ、ジャワのガムラン、バリのコテカン、インドのターラ、バルカンのアクサクの伝統に関する民族音楽学研究を、パラメータ化された解釈可能な生成システムに統合している。本論文では、フレームワークのコアアーキテクチャ、アルゴリズムプロセス、専門エンジン、ヒューマナイゼーションメカニズム、および理論的基盤について説明する。

1. 序論

1.1 背景

計算リズム生成は主に西洋のポピュラー音楽スタイルに焦点を当ててきた。Groove MIDIデータセットやMagentaのドラム転写コーパスなどの主要な公開データセットは、ロック、ポップ、ファンク、ジャズを含む西洋のジャンルでほぼ完全に構成されている。最近の研究（Mehta et al., 2024）は100万時間以上のオーディオデータセットを分析し、86%がグローバルノースの音楽に集中し、93%の研究者が主に西洋音楽を研究していることを発見した。GrooVAEやDrum RNNなどの機械学習アプローチは、表現力豊かなドラム演奏生成で成功しているが、主にこれらの西洋中心のデータセットで訓練されており、文化固有のリズム原則の明示的なモデリングが欠けている。

リズムは音楽の基礎要素であり、世界各地の音楽伝統は独自のリズムシステムを発展させてきた。西アフリカのポリフォニック打楽器、キューバのClaveシステム、ブラジルのサンバグルーヴから、ジャワガムランのColotomic構造まで、これらのリズム伝統はそれぞれ独自の時間組織原理と美的概念を持っている。

グローバリゼーションと電子音楽の発展に伴い、異文化リズムの融合は現代音楽創作の重要なトレンドとなっている。しかし、各伝統の特性を維持しながら有機的な融合を実現する方法は、クリエイターが直面する課題であり続けている。従来のドラムマシンやリズムプログラミングツールは西洋の4/4拍子を基礎としており、非西洋リズムの微妙な質を自然に表現することが難しい。

1.2 研究目的

本研究は以下を可能にするアルゴリズムシステムを開発することを目的とする：

  1. 10の主要な世界音楽リズムスタイルのコア特性を忠実に表現
  2. 単一スタイル内でのバリエーション生成をサポート
  3. スタイル固有の特性を維持しながら全体的な調和を保つインテリジェントなクロススタイルミキシングを実現
  4. 自然な演奏特性に合わせたヒューマナイゼーション処理を提供
  5. Articulation Profileシステムを通じて文化固有の装飾技法を実装

2. 関連研究

2.1 異文化リズム研究

Simha Arom（1991）は、中央アフリカ共和国のBanda Linda族およびAka Pygmiesの音楽研究において、アフリカのポリフォニック打楽器の複雑なインターロック構造を明らかにした。彼が提唱した「最小周期」の概念は、非西洋リズムを分析する重要なツールとなった。注目すべきは、Aromの研究は主に中央アフリカ（Central Africa）地域に焦点を当てており、タイムラインパターン（timeline patterns）が最も顕著な西アフリカ（West Africa）地域ではないことである。これら2つの地域のリズム伝統は共通原則を持つが、具体的な実践においては異なる。

Gerhard Kubik（2010）は、アフリカリズムの「タイムライン」（timeline）概念をさらに詳述した。これは、アンサンブル全体に浸透する参照リズムパターンであり、他のパートはこのタイムラインを中心に組織される。この概念は、本研究の役割階層アーキテクチャに直接的なインスピレーションを与えた。

Fernando Benadon（2006）のジャズマイクロタイミング研究は、人間の演奏におけるタイミングのずれはランダムなエラーではなく、スタイル固有の表現手段であることを示した。これは、本研究のヒューマナイゼーションメカニズムの理論的基盤を提供した。

2.2 アルゴリズムリズム生成

Godfried Toussaint（2013）の研究は、ユークリッドアルゴリズムと世界音楽リズムの間の数学的関連を確立し、多くの伝統的リズムパターンがユークリッドアルゴリズムの特定パラメータで生成できることを発見した。本研究は、この方法を基本リズム生成のコアとして採用している。

Jeff Pressing（1983）は、リズムの難易度を定量化するための認知複雑度モデルを提唱した。本研究は、密度制御とバリエーション生成においてこの概念を参照している。

2.3 既存システムの限界

Ableton LiveのBeat RepeatやNative InstrumentsのMaschineなどの既存のリズム生成ツールは強力だが、主に電子音楽とヒップホップスタイル向けに設計されている。それらには以下が欠けている：

  • 非西洋リズムのタイムライン概念のサポート
  • スタイル固有のスウィングとマイクロタイミング処理
  • 異文化スタイルミキシングのためのインテリジェントなメカニズム

4. コアアルゴリズム

4.1 ユークリッドリズム生成

基本リズムパターンは、Bjorklund（2003）のユークリッドアルゴリズムを使用して生成される。総ステップ数nとヒット数kが与えられると、アルゴリズムはnステップ内のk個のヒットの最大均等分布を生成する：

std::vector<bool> euclidean(int hits, int steps) {
    std::vector<int> pattern(steps, 0);
    int bucket = 0;

    for (int i = 0; i < steps; i++) {
        bucket += hits;
        if (bucket >= steps) {
            bucket -= steps;
            pattern[i] = 1;
        }
    }
    return pattern;
}

このアルゴリズムは多くの伝統的リズムパターンを生成できる：

  • E(3,8) = Tresillo（キューバ）
  • E(5,8) = Cinquillo（キューバ）
  • E(7,12) = 西アフリカ標準ベルリズム
  • E(5,16) = Bossa Nova

4.2 インターロック生成アルゴリズム

インターロック（Interlocking）は、マルチパートの調和を確保する重要なメカニズムである。システムは3つのインターロック戦略を実装する：

回避戦略：FoundationはTimelineの強拍位置を避ける

float calculateAvoidance(int step, const Pattern& timeline,
                         float avoidanceStrength) {
    if (timeline.hasHit(step)) {
        // Timelineにヒットがある場合、この位置の確率を減少
        return 1.0f - avoidanceStrength;
    }
    return 1.0f;
}

補完戦略：GrooveはTimelineとFoundationの間のギャップを埋める

float calculateComplement(int step, const Pattern& timeline,
                          const Pattern& foundation, float boost) {
    bool hasTimelineHit = timeline.hasHit(step);
    bool hasFoundationHit = foundation.hasHit(step);

    if (!hasTimelineHit && !hasFoundationHit) {
        // 空白位置はブーストを受ける
        return boost;
    }
    return 1.0f;
}

交渉戦略：Leadボイスは他のすべてのボイスを考慮

float calculateLeadProbability(int step, const Pattern patterns[3],
                               float baseProb) {
    int activeVoices = 0;
    for (int i = 0; i < 3; i++) {
        if (patterns[i].hasHit(step)) activeVoices++;
    }

    // アクティブなボイスが多いほど、Leadは休む傾向
    return baseProb * (1.0f - activeVoices * 0.2f);
}

互換性は以下の原則に基づいて計算される：

  1. パルスファミリー：同じ基本細分を共有するスタイルは互換性が高い（例：12/8ファミリー：西アフリカ、キューバ、ブラジル）
  2. 歴史的関連：歴史的つながりのあるスタイルは互換性が高い（例：キューバとジャズ）
  3. タイミング特性：類似したヒューマナイゼーション度のスタイルは混合しやすい（例：エレクトロニックとテクノ）

互換性が低い場合、システムは自動的にインターロックを強化する：

float getInterlockStrength(int styleA, int styleB) {
    float compatibility = getCompatibility(styleA, styleB);
    // 互換性が低いほど、インターロックが強くなり、全体的な調和を維持
    return 1.0f + (1.0f - compatibility) * 0.5f;
}

4.4 ヒューマナイゼーション処理

ヒューマナイゼーションエンジンは、人間の演奏におけるタイミングとベロシティの変動をシミュレートし、スタイル固有の特性を持つ：

タイミング変動

float getTimingOffset(int step, int styleIndex, float amount) {
    const StyleTimingProfile& profile = profiles[styleIndex];

    // 基本分散
    float variance = profile.baseVariance * amount;

    // 役割補正（Timelineが最も安定、Leadが最も自由）
    variance *= profile.roleMultipliers[currentRole];

    // ガウス分布のオフセットを生成
    return gaussianRandom(0, variance);
}

BPM依存スウィング

Friberg & Sundström（2002）の研究によると、ジャズのスウィング比率はテンポによって変化する：

  • 低速（~120 BPM）：最大3.5:1
  • 中速：約2:1（三連符感、67%）
  • 高速（300+ BPM）：1:1に近づく（ストレート）

この現象は生体力学的制限に由来する——高速では大きな不均等細分を維持することが困難である。

float getSwingRatio(int styleIndex, float bpm) {
    const StyleTimingProfile& profile = profiles[styleIndex];

    float slowRatio = profile.swingRatioSlow;  // 低速時のスウィング
    float fastRatio = profile.swingRatioFast;  // 高速時のスウィング

    // 線形補間、100-180 BPMが遷移ゾーン
    float t = clamp((bpm - 100.0f) / 80.0f, 0.0f, 1.0f);
    return lerp(slowRatio, fastRatio, t);
}

相対的ゴーストノートベロシティ

ゴーストノート（ghost notes）のベロシティは、絶対値ではなく前のメイン音に対して相対的に計算される。学術研究によると：

  • Matsuo & Sakaguchi（2024）：1:4振幅比 = 25%
  • Cheng et al.（2022）：10dB差 = ~32%

したがって、ゴーストノートベロシティ範囲は25-32%に設定される。

float getGhostVelocity(float previousVelocity, int styleIndex) {
    const StyleTimingProfile& profile = profiles[styleIndex];
    // ghostVelocityMin = 0.25, ghostVelocityMax = 0.32 (査読済み)
    float ratio = randomRange(profile.ghostVelocityMin,
                              profile.ghostVelocityMax);
    return previousVelocity * ratio;
}

4.5 アーティキュレーション・プロファイル・システム

スタイル固有の装飾技法を実現するため、システムは民族音楽学研究に基づくArticulation Profile参照テーブルを採用している。各スタイルと役割の組み合わせには専用の装飾技法設定がある。

装飾技法タイプ

enum class ArticulationType {
    None,   // 装飾なし
    Flam,   // ダブルストライク（前打音）
    Drag,   // ダブル前打音
    Ruff,   // トリプル前打音
    Buzz    // バズロール（連続細かいストローク）
};

プロファイル構造

struct ArticulationEntry {
    ArticulationType type;      // 装飾技法タイプ
    float baseProbability;      // 基本発生確率
    bool onAccentsOnly;         // アクセント時のみトリガー
    bool onStrongBeats;         // 強拍時のみトリガー
};

// 10スタイル × 4役割 = 40プロファイルセット
static const ArticulationEntry profiles[10][4];

選択アルゴリズム

ArticulationType selectArticulation(int styleIndex, int roleIndex,
                                    float amount, bool isAccent,
                                    bool isStrongBeat) {
    const ArticulationEntry& entry = profiles[styleIndex][roleIndex];

    // 条件チェック
    if (entry.onAccentsOnly && !isAccent) return None;
    if (entry.onStrongBeats && !isStrongBeat) return None;

    // 確率計算：基本確率 × ユーザー制御量
    float probability = entry.baseProbability * amount;

    if (randomFloat() < probability) {
        return entry.type;
    }
    return None;
}

6.3 限界

  1. 文化的簡略化：複雑な音楽伝統をパラメータに簡略化することで、不可避的に一部の詳細が失われる
  2. 静的互換性：互換性マトリクスはプリセット値であり、特定の音楽コンテキストを反映できない
  3. ピッチ次元の欠如：システムはリズムに焦点を当てており、メロディとハーモニーの異文化融合を扱っていない

7. 結論と今後の課題

7.1 結論

本研究で提案したWorldRhythmアルゴリズムは以下を成功裏に実現した：

  1. マルチスタイルサポート：10の主要世界音楽リズム伝統をカバー
  2. インテリジェントミキシング：互換性マトリクスと可変強度インターロックを通じてクロススタイル融合を実現
  3. ヒューマナイゼーション処理：スタイル固有のタイミングとベロシティ変動により音楽性を向上
  4. Articulation Profile：民族音楽学に基づく装飾技法システムで、各スタイル×役割の組み合わせに専用設定
  5. モジュラーアーキテクチャ：新しいスタイルと機能の拡張が容易

このシステムは、各伝統への敬意と理解を維持しながら、音楽クリエイターに異文化リズム融合を探求するツールを提供する。

7.2 今後の課題

  1. 機械学習統合：実際の演奏データを使用してスタイルモデルを訓練し、リアリズムを向上
  2. 動的互換性：実際の生成結果に基づいて互換性判断を動的に調整
  3. ユーザー研究：リスニングテストを実施し、生成されたリズムの音楽性と文化的適切性を評価
  4. ピッチ拡張：メロディとハーモニーの異文化生成能力を追加
